    Introduction

    Welcome to the TRUEPrice™ API! This document describes how to interact with TRUEPrice™ API services to retrieve TLE product.

    Contact TLE technical support for more information.

    • Visit Our Website!

    Prerequisites:

    • Ensure you have Python 3 installed.
    • Install the requests library if it’s not already installed: pip install requests

    • You will need API credentials (email and password) to authenticate and obtain an access token for secure requests.

    • Contact TLE technical support at techsupport@truelightenergy.com for more information.

    Authentication

    To authorize, use this code:

    import requests
def login(email = "email@forTRUEPrice™.com", password = "password"):
             url = "https://truepriceenergy.com/login"
             querystring = {"email":email,"password":password}
             response = requests.request("GET", url, params=querystring, verify=False)
             return response.text

#Example Usage
response = login("your_email@example.com", "your_password")

    Make sure to replace email & password with your own credentials.

    • Ensure you have imported the necessary Python modules at the start of your script: import requests
    • The login function authenticates the user using their email and password, returning a login response that includes an access token. This token is necessary for subsequent API calls.
    • Default Credentials: You may pass the default credentials "dummy@truelightenergy.com" or your own credentials.

    Output: The function returns the access token (or error response)

    You must replace email & password with your own credentials.

    Functions

    Download Data

    The Access Token recieved after login is used in this function to download data.

    def get_data(access_token, start_date, end_date, operating_day, operating_day_end, curve, iso, strip, idcob, type):
  url = "https://truepriceenergy.com/get_data"
  querystring = 
  {
    "start":start_date,
    "end":end_date,
    "curve_type": curve,
    "iso":iso,
    "strip": strip,
    "idcob":idcob,
    "type": type,
    "operating_day":operating_day,"operating_day_end":operating_day_end
    }
  headers = {"Authorization": f"Bearer {access_token}"}
  response = requests.request("GET", url, params=querystring, headers=headers, verify=False)


  with open(f"{curve}_{iso}.csv", "wb") as file:
        file.write(response.content)
  return response


# Example:
access_token = eval(login("your_email@example.com", "your_password"))["access_token"]

#Response
data_response = get_data(your_access_token, your_start_date, your_end_date, your_operating_day,your_operating_day_end, your_curve, your_iso, your_strip, your_idcob, your_type)

    HTTP Request

    GET https://truepriceenergy.com/get_data

    Query Parameters

    Parameter Description
    access_token The token received from the login function.
    start_date Start date in YYYY-MM-DD format.
    end_date End date in YYYY-MM-DD format.
    operating_day The operating day for which data is required (optional). YYYY-MM-DD format.
    operating_day_end The operating day end for which data is required (optional) YYYY-MM-DD format.
    curve Type of curve (e.g., "energy").
    iso ISO specification (e.g., "ercot", "nyiso").
    strip Format (standardized,unstandardized ).
    idcob String indicating whether to include historical data.(latestall,intradayonly)
    type Format for data retrieval ("csv",xlsx, "json").

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The TruePrice API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The request is hidden for administrators only.
    404 Not Found -- The specified request could not be found.
    405 Method Not Allowed -- You tried to access with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The request has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many files! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
